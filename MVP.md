# Continuous Crossword - MVP 需求文档

## 项目概述

**项目名称**: Continuous Crossword  
**目标**: 开发一个连续填字游戏网站，用户根据线索猜单词，答对后立即进入下一题  
**技术栈**: Next.js + Datamuse API + Cloudflare Pages

---

## 核心功能（MVP 范围）

### 1. 游戏主界面
**优先级**: P0（必须）

**功能描述**:
- 显示当前线索（clue）
- 显示答案字母数提示
- 提供输入框供用户输入答案
- 提交按钮

**验收标准**:
- 界面简洁清晰
- 响应式设计（支持手机/平板/桌面）
- 输入框自动聚焦

---

### 2. 答案验证
**优先级**: P0（必须）

**功能描述**:
- 用户提交答案后，系统验证正确性
- 正确：显示成功提示，1秒后自动加载下一题
- 错误：显示错误提示，允许继续尝试

**验收标准**:
- 不区分大小写
- 去除首尾空格
- 即时反馈（无需刷新页面）

---

### 3. 计分系统
**优先级**: P0（必须）

**功能描述**:
- 答对一题 +10 分
- 显示当前总分
- 显示连续答对次数（streak）
- 答错时 streak 归零

**验收标准**:
- 分数实时更新
- 页面刷新后分数重置（MVP 阶段不持久化）

---

### 4. 题库接口
**优先级**: P0（必须）

**功能描述**:
- 使用 Datamuse API 获取单词和定义
- 随机选择常见英文单词
- 返回线索（定义）和答案

**API 设计**:
```
GET /api/clue
Response: {
  clue: string,      // 线索文本
  answer: string,    // 正确答案
  length: number     // 答案字母数
}
```

**验收标准**:
- API 响应时间 < 2秒
- 有备用线索机制（API 失败时）
- 单词难度适中（常见词汇）

---

## 非功能需求

### 性能
- 首屏加载时间 < 3秒
- 答案验证响应 < 100ms

### 兼容性
- 支持 Chrome/Safari/Firefox 最新版本
- 支持 iOS Safari 和 Android Chrome

### 可用性
- 无需注册/登录
- 无需安装
- 打开即玩

## 技术实现细节

### 前端状态管理
使用 React useState 管理：
- 当前题目数据
- 用户输入
- 分数和连胜数
- 加载状态
- 提示消息

### API 调用流程
1. 组件挂载时调用 `/api/clue`
2. 显示线索和输入框
3. 用户提交答案
4. 验证通过后再次调用 `/api/clue`

### 样式设计
- 主色调：蓝色/紫色渐变
- 卡片式布局
- 圆角、阴影提升质感
- 动画反馈（成功/失败）

## 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| Datamuse API 不稳定 | 无法获取题目 | 准备备用题库（JSON 文件） |
| API 响应慢 | 用户体验差 | 添加加载动画，超时重试 |
| 单词太难 | 用户挫败感 | 精选常见词汇列表 |
